# Parser conformance corpus for internal/jenkinsurl.Parse.
#
# These URLs were collected from production Jenkins instances and the
# e2e harness; each one is exercised by Test_Parse in
# internal/jenkinsurl/parse_test.go. Comment lines (starting with `#`)
# and blank lines are ignored.
#
# The corpus intentionally covers every documented shape:
#   - top-level single-segment pipeline (with/without trailing slash)
#   - nested folder paths
#   - multibranch (branch is a regular /job/ segment)
#   - build-number-suffixed
#   - non-default port (HTTP 8080)
#   - explicit default ports (:443, :80) that must be normalized away
#   - percent-encoded path segments
#   - mixed case hostname (must lowercase)
#   - trailing query + fragment (must be stripped from JobSegments)
#   - localhost harness URLs
#   - context-path-mounted instances (prefix before the first /job/)

# Top-level pipeline, trailing slash
https://jenkins.foo.com/job/svc/

# Top-level pipeline, no trailing slash
https://jenkins.foo.com/job/svc

# Folder-nested pipeline (two folders deep)
https://jenkins.foo.com/job/team/job/platform/job/svc/

# Multibranch pipeline build (folder + multibranch + branch + #42)
https://jenkins.foo.com/job/team/job/svc/job/main/42/

# Non-default port (e2e harness shape)
http://jenkins.local:8080/job/svc/

# Explicit default HTTPS port (normalize away :443)
https://jenkins.foo.com:443/job/svc/

# Explicit default HTTP port (normalize away :80)
http://jenkins.foo.com:80/job/svc/

# Percent-encoded segment (`my pipeline` with a space)
https://jenkins.foo.com/job/my%20pipeline/

# Query + fragment must not leak into JobSegments
https://jenkins.foo.com/job/svc/?foo=bar#frag

# Build-number suffix without trailing slash
https://jenkins.foo.com/job/svc/7

# Mixed-case hostname (must lowercase to canonical Host)
https://Jenkins.FOO.com/job/svc/

# Local e2e harness build URL (deploy-input pipeline)
http://localhost:18080/job/deploy-input/21/

# --- Context-path-mounted instances (parse-context-path-prefix) ---
# Jenkins behind a reverse proxy or multi-tenant mount lives under a context
# path that precedes the first /job/ segment. The prefix is captured on
# Ref.BasePath (verbatim, leading slash, no trailing slash) and emitted by
# APIPath before /job/. It does NOT affect the host credential key. A path
# with no /job/ token anywhere is still rejected.

# Single-segment context path with build number (BasePath "/domain")

# Common reverse-proxy /jenkins mount on a bare job (BasePath "/jenkins")
https://ci.example.com/jenkins/job/svc/

# Multi-segment context path (BasePath "/team/ci")
https://example.com/team/ci/job/svc/job/main/42/

# Context path with a permalink trailing segment
https://example.com/jenkins/job/svc/lastSuccessfulBuild/
